# red-green：Claude Code 桌面宠物

让用户在终端不在前台时，仍能及时感知本机 Claude Code 会话的状态。感知的载体是一只桌面宠物：一个常驻屏幕的环境状态指示器，用行为动画表达会话状态，而不是用弹窗打断用户。

## Language

**宠物 (Pet)**:
常驻屏幕、置顶于所有窗口的动画小生物，是会话状态的可视化身。
_Avoid_: 挂件、widget、指示器

**皮肤 (Skin)**:
宠物的视觉形象与叫声音色，现有三套：灯灯（红绿灯头机器人）、钳钳（螃蟹）、灰灰（猫）。按项目名确定性分配——同一项目永远同一只，物种本身即会话身份的一部分，看形象、听音色皆可辨识项目。
_Avoid_: 主题、造型

**气泡 (Bubble)**:
悬在宠物头顶的状态图示：注意力事件与空闲显示表意符号，运行中显示进行中的动词。它是宠物话语的一部分——紧贴宠物当前姿态的头顶，随姿态起落、随动作同行，不是窗口里位置固定的标牌。
_Avoid_: emoji（那只是内容之一，不是这个元素）、提示框、通知

**会话状态 (Session State)**:
一个 Claude Code 会话在任意时刻所处的状态：运行中、待确认、轮到你、已完成、异常中止、空闲。宠物的行为动画由它驱动。
_Avoid_: 进度、阶段

**运行中 (Running)**:
Claude Code 正在执行一个回合，不需要用户做任何事。
_Avoid_: 忙碌、处理中

**注意力事件 (Attention Event)**:
Claude Code 会话中值得把用户的注意力拉回来的时刻。分为三个子类：待确认、已完成、异常中止。
_Avoid_: 通知（那是对事件的响应，不是事件本身）、消息

**待确认 (Awaiting Confirmation)**:
回合进行中，Claude Code 被一个必须由用户处理的弹窗卡住——权限确认、弹窗提问。任务真正被阻塞，是最高紧急度。回合结束后的等待不属于此状态（那是"轮到你"）。
_Avoid_: 等待、卡住、提问

**轮到你 (Your Turn)**:
回合以向用户提问收尾，Claude 在等答复才能继续。由回合最后一条消息的内容判定（问句启发式）。不紧急但持续成立——直到用户真正作答才解除，不因已阅而消失。
_Avoid_: 等输入、空闲等待（它不是空闲，Claude 有未决问题）

**已完成 (Completed)**:
一个回合正常结束，Claude Code 交还控制权。
_Avoid_: 结束、done

**异常中止 (Aborted)**:
回合因外部错误（限流、账单、服务器错误）非正常终止，任务没有做完也不会自行恢复。
_Avoid_: 失败（易与"任务结果不对"混淆）、报错

**空闲 (Idle)**:
会话存活但无事发生：回合已结束且结果已被已阅，或尚未开始新回合。宠物睡觉。
_Avoid_: 挂起、暂停

**叫声 (Call)**:
宠物在会话状态转移瞬间发出的一次性声音。音色由物种决定（听声辨项目），节奏由进入的状态决定（听节奏辨事件）：待确认=急促三连音、异常中止=下坠双音、轮到你=上扬双音、已完成=轻短单音；响度随紧急度递减，已完成最轻。运行中与空闲无叫声。
_Avoid_: 音效、提示音、通知（那是系统弹窗体系的词）

**已阅 (Acknowledged)**:
用户把某会话的终端切到前台这一动作，视为已经看到了该会话的结果。已完成与异常中止在已阅后转为空闲；轮到你不适用已阅——看到了问题不等于回答了问题。
_Avoid_: 确认（已被"待确认"占用，语义不同）、dismiss

**前台静默 (Foreground Suppression)**:
某会话自己的终端标签页位于前台时，该会话的状态转移不发出叫声的规则。用户正在看着，就不出声；被静默的叫声就此消失，不在事后补发。宠物的动画不受此规则影响，始终如实反映状态。其他会话的叫声照常。
_Avoid_: 免打扰（那是系统级勿扰模式，不是这个规则）
